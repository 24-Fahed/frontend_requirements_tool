# 需求收集工具 - 子系统接口文档

## 1. 概述

本文档定义系统三个子系统之间的主要接口与调用边界。

## 2. 子系统接口图

![子系统接口图](images/子系统接口图.png)

## 3. 接口总览

| 接口标识 | 接口名称 | 提供方 | 调用方 | 调用方式 |
|----------|----------|--------|--------|----------|
| `I_Navigation` | 导航接口 | 界面子系统 | 需求收集子系统（前端） | Store / 组件事件 |
| `I_Layout` | 宿主布局接口 | 界面子系统 | 交互式需求收集子系统 | Store |
| `I_Simulator` | 模拟器接口 | 交互式需求收集子系统 | 需求收集子系统（前端） | Store / 组件调用 |
| `I_ComponentRegistry` | 组件注册接口 | 交互式需求收集子系统 | 后端静态配置 | HTTP + 前端运行时加载 |
| `I_LayoutRegistry` | 布局注册接口 | 交互式需求收集子系统 | 后端静态配置 | HTTP |
| `I_ParseService` | 配置解析接口 | 后端 | 前端 | HTTP GET |
| `I_StorageService` | 结果存储接口 | 后端 | 前端 | HTTP POST / GET |
| `I_StaticService` | 静态资源接口 | 后端 | 交互式需求收集子系统 | HTTP GET |

## 4. 前端内部接口

### 4.1 `I_Navigation`

提供方：界面子系统  
调用方：需求收集子系统（前端）

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `switchCategory` | `categoryId: string` | `void` | 切换左侧需求分类 |

说明：

- `v0.2.0` 起，`I_Navigation` 不再暴露组件面板开关。
- 组件面板开关属于交互式编辑器内部行为，由模拟器状态栏控制。

### 4.2 `I_Layout`

提供方：界面子系统  
调用方：交互式需求收集子系统

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `setSimulatorVisible` | `visible: boolean` | `void` | 控制模拟器宿主区域显隐 |
| `setContentData` | `data: object` | `void` | 为内容区域提供当前需求数据 |

### 4.3 `I_Simulator`

提供方：交互式需求收集子系统  
调用方：需求收集子系统（前端）

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `loadDefault` | `layoutId: string` | `void` | 加载默认布局 |
| `openForQuestion` | `questionId: string, defaultLayoutId?: string` | `void` | 打开指定交互式问题并重置编辑器状态 |
| `getResult` | `-` | `{ nodes: LayoutNode[] }` | 获取当前模拟器设计结果 |
| `reset` | `-` | `void` | 重置模拟器状态 |

内部说明：

- `PhoneFrame` 作为宿主组件创建独立 iframe document。
- `SimulatorDocumentApp` 在 iframe 中运行，但与主应用共享 Pinia store。
- `previewMode`、`paletteVisible` 等编辑器内部状态属于交互式子系统，不再暴露给界面子系统。

### 4.4 `I_ComponentRegistry`

提供方：交互式需求收集子系统  
调用方：系统初始化流程 / 后端配置

| 方法 / 数据 | 返回值 | 说明 |
|-------------|--------|------|
| `loadComponents()` | `Promise<void>` | 加载组件与布局元数据 |
| `componentList` | `ComponentInfo[]` | 非容器组件列表 |
| `layoutList` | `ComponentInfo[]` | 容器布局列表 |

### 4.5 `I_LayoutRegistry`

提供方：交互式需求收集子系统  
调用方：系统初始化流程 / 后端配置

| 方法 / 数据 | 返回值 | 说明 |
|-------------|--------|------|
| `loadDefaultLayouts()` | `Promise<void>` | 加载默认布局配置 |
| `defaultLayouts` | `LayoutScheme[]` | 默认布局方案列表 |

## 5. 后端 HTTP 接口

### 5.1 `I_ParseService`

提供方：后端  
调用方：前端

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/requirements` | 获取需求配置 |
| `GET` | `/api/components` | 获取组件元数据 |
| `GET` | `/api/default-layout` | 获取默认布局配置 |

### 5.2 `I_StorageService`

提供方：后端  
调用方：前端

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/results` | 保存需求结果 |
| `GET` | `/api/results` | 获取已保存结果 |

### 5.3 `I_StaticService`

提供方：后端  
调用方：交互式需求收集子系统

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/static/lib/{path}` | 获取组件库中的 `.vue` 文件 |

## 6. 数据类型摘要

### `LayoutNode`

```ts
type LayoutNode = {
  id: string
  type: string
  props: Record<string, any>
  children?: LayoutNode[]
}
```

### `ComponentInfo`

```ts
type ComponentInfo = {
  name: string
  label: string
  category: string
  icon?: string
  props: PropDef[]
  defaultProps: Record<string, any>
  sourceFile?: string
  componentTag?: string
  isContainer?: boolean
  maxChildren?: number
}
```

## 7. v0.2.0 接口变化

1. 删除 `I_Navigation.togglePanel`。
2. 保留 `I_Simulator` 的核心方法签名，不修改结果结构。
3. 组件面板显隐状态从 `appStore.panelOpen` 收回到交互式子系统内部。
4. 界面子系统只保留宿主级接口，不再控制编辑器内部行为。
