# 需求收集工具 - 部署手册

## 1. 概述

本文档指导实施工程师完成需求收集工具的部署，包括本地开发部署和服务器生产部署。

所有配置通过修改 `.env` 文件管理，无需修改任何代码文件。

## 2. 目录结构

```
require_tools/
├── docker/                      # Docker 部署文件
│   ├── Dockerfile               # 镜像构建文件
│   ├── docker-compose.yml       # 容器编排文件
│   ├── .env.local               # 本地环境配置
│   ├── .env.prod                # 生产环境配置
│   ├── start.sh                 # 启动脚本（Linux）
│   └── stop.sh                  # 停止脚本（Linux）
├── data/                        # 数据目录（Volume 挂载）
│   ├── dev-config/              # 开发者配置文件（输入）
│   │   ├── requirements.yaml    # 需求问题定义
│   │   ├── components.json      # 组件描述
│   │   ├── default_layout.json  # 默认布局方案
│   │   └── lib/                 # 组件库文件
│   │       └── your-lib.js          # 开发者自定义的组件库
│   └── output/                  # 用户需求结果（输出）
│       ├── result.yaml
│       └── result.json
├── backend/                     # 后端源码
├── frontend/                    # 前端源码
└── docs/                        # 设计文档
```

## 3. 环境要求

| 环境 | 版本要求 |
|------|----------|
| Docker | 20+ |
| Docker Compose | V2（docker compose 命令） |

无需安装 Python、Node.js，所有依赖在 Docker 镜像内构建。

## 4. 配置说明（.env 文件）

所有参数通过 `.env` 文件管理。工具提供两个预设配置：

| 文件 | 用途 |
|------|------|
| `.env.local` | 本地开发/测试 |
| `.env.prod` | 生产服务器部署 |

### 配置项说明

```bash
# ---- 运行环境 ----
APP_ENV=local                    # local 或 production

# ---- 镜像源（默认国内镜像，通常无需修改）----
PYTHON_BASE_IMAGE=docker.m.daocloud.io/library/python:3.10-slim
NODE_BASE_IMAGE=docker.m.daocloud.io/library/node:20-alpine
PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
PIP_TRUSTED_HOST=pypi.tuna.tsinghua.edu.cn
NPM_REGISTRY=https://registry.npmmirror.com

# ---- 服务端口 ----
HOST=0.0.0.0                     # 容器内监听地址
PORT=8000                        # 容器内服务端口
BIND_PORT=8000                   # 宿主机映射端口（生产环境改为80）

# ---- 数据卷路径（相对于 docker-compose.yml 所在目录）----
DEV_CONFIG_DIR=../data/dev-config    # 开发者配置文件路径
OUTPUT_DIR=../data/output            # 需求结果输出路径
```

### 生产环境关键配置

生产部署时，只需修改 `.env.prod` 中的以下项：

```bash
APP_ENV=production
BIND_PORT=80                     # 改为80端口对外服务
```

## 5. 本地开发部署

### 5.1 启动服务

**Linux / macOS：**
```bash
cd docker
chmod +x start.sh stop.sh
./start.sh local
```

**Windows（PowerShell）：**
```powershell
cd docker
docker compose --env-file .env.local -f docker-compose.yml up -d --build
```

启动后访问 `http://127.0.0.1:8000`

### 5.2 准备配置文件

将开发者准备的配置文件放入 `data/dev-config/` 目录：

```
data/dev-config/
├── requirements.yaml      # 必需
├── components.json         # 必需
├── default_layout.json     # 可选
└── lib/                    # 必需（组件库目录）
    └── your-lib.js            # 开发者自定义组件库
```

配置文件格式请参考 `HELP.md`（工程师使用手册）。

### 5.3 获取需求结果

用户填写完需求后点击"保存结果"，结果自动写入 `data/output/` 目录：

```
data/output/
├── result.yaml          # 人类可读
└── result.json          # 结构化数据
```

### 5.4 停止服务

```bash
./stop.sh
```

或 Windows：
```powershell
docker compose -f docker-compose.yml down
```

## 6. 服务器生产部署

### 6.1 上传项目

将整个 `require_tools/` 目录上传到服务器，例如 `/opt/require-tools/`：

```bash
scp -r require_tools/ user@your-server:/opt/require-tools/
```

### 6.2 修改生产配置

编辑 `docker/.env.prod`：

```bash
APP_ENV=production
BIND_PORT=80                     # 对外使用80端口
# 其他项保持默认即可
```

### 6.3 准备配置文件

将工程师准备的配置文件放入 `data/dev-config/`：

```bash
# 上传配置文件
scp requirements.yaml user@your-server:/opt/require-tools/data/dev-config/
scp components.json user@your-server:/opt/require-tools/data/dev-config/
scp -r lib/ user@your-server:/opt/require-tools/data/dev-config/lib/
```

### 6.4 启动服务

```bash
cd /opt/require-tools/docker
chmod +x start.sh stop.sh
./start.sh prod
```

启动后，通过服务器IP即可访问：`http://your-server-ip`

### 6.5 获取需求结果

```bash
# 从服务器下载结果
scp user@your-server:/opt/require-tools/data/output/result.yaml ./
scp user@your-server:/opt/require-tools/data/output/result.json ./
```

### 6.6 停止服务

```bash
./stop.sh
```

## 7. 更新配置文件

配置文件更新后无需重新构建镜像，只需：

```bash
# 1. 替换 data/dev-config/ 中的文件
scp new-requirements.yaml user@server:/opt/require-tools/data/dev-config/requirements.yaml

# 2. 重启容器使配置生效
cd /opt/require-tools/docker
docker compose -f docker-compose.yml restart
```

如果更新了后端或前端代码，则需要重新构建：

```bash
./start.sh prod
```

## 8. 常用操作

| 操作 | 命令 |
|------|------|
| 查看运行状态 | `docker compose -f docker-compose.yml ps` |
| 查看日志 | `docker logs require-tools` |
| 实时查看日志 | `docker logs -f require-tools` |
| 重启服务 | `docker compose -f docker-compose.yml restart` |
| 进入容器调试 | `docker exec -it require-tools sh` |
| 清理并重建 | `docker compose -f docker-compose.yml down && ./start.sh prod` |

## 9. 数据卷说明

| 本地路径 | 容器路径 | 说明 |
|----------|----------|------|
| `data/dev-config/` | `/app/dev-config` | 开发者配置文件（只读建议） |
| `data/output/` | `/app/output` | 用户需求结果（写入） |

这两个目录通过 Volume 挂载，容器删除后数据不会丢失。工程师可以直接在本地文件系统中操作这些文件。

## 10. 防火墙配置

生产服务器需要开放对应端口：

```bash
# 开放80端口
sudo ufw allow 80
# 或
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```
