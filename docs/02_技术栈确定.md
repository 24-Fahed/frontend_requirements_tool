# 需求收集工具 - 技术栈确定

## 1. 前端技术栈

| 类别 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 框架 | Vue.js | 3.x (Composition API) | 用户指定，使用组合式API |
| 构建工具 | Vite | 5.x | Vue 3 官方推荐，开发体验好 |
| 工具界面组件库 | Element Plus | 2.x | 仅用于工具自身界面（工具栏、侧边栏、抽屉面板、表格等） |
| 手机模拟器 | 自定义Vue组件 + CSS | - | 纯CSS实现手机外壳外观；内部使用Vue动态组件 `<component :is="...">` 渲染开发者组件；配合vuedraggable实现拖拽交互 |
| 模拟器组件 | 开发者自定义 | - | 开发者导入自有UI组件库，提供组件定义文件；工具动态加载并渲染 |
| 拖拽库 | vuedraggable | 4.x | 基于 Sortable.js，支持 Vue 3，用于组件/布局拖拽 |
| 状态管理 | Pinia | 2.x | Vue 3 官方推荐状态管理 |
| 路由 | Vue Router | 4.x | Vue 3 配套路由 |
| HTTP客户端 | Axios | 1.x | 与后端 API 通信 |

## 2. 后端技术栈

| 类别 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 语言 | Python | 3.10+ | 用户指定 |
| Web框架 | FastAPI | 0.100+ | 用户指定，异步高性能，自带OpenAPI文档 |
| ASGI服务器 | Uvicorn | 0.23+ | FastAPI 官方推荐 |
| 数据解析 | Pydantic | 2.x | FastAPI 内置，用于请求/响应数据验证 |
| 文件处理 | PyYAML | 6.x | 解析开发者编写的 YAML 格式配置文件 |
| 数据序列化 | 内置 json / yaml | - | 输出用户需求结果文件 |

## 3. 数据格式选型

| 用途 | 格式 | 说明 |
|------|------|------|
| 开发者需求配置文件 | YAML | 可读性好，支持注释，适合开发者手动编写 |
| 开发者组件/布局定义 | JSON | 结构化强，前端可直接解析使用 |
| 交互式需求结果 | JSON | 结构化数据，便于程序处理 |
| 最终需求输出文件 | YAML + JSON | YAML用于人类可读的汇总，JSON用于结构化数据交换 |

## 4. 模拟器组件机制

模拟器中的UI组件不由工具绑定，而是由开发者提供。具体机制：

- **开发者职责**：提供组件库（如 .js/.vue 编译后的文件）+ 组件描述文件（JSON），描述每个组件的名称、属性、默认值
- **工具职责**：读取组件描述文件，在抽屉面板中展示组件列表；拖拽到模拟器后，动态加载并渲染对应的组件
- **组件类型**：基本组件（按钮、文本框等）和组合组件（多个基本组件封装为一个 component）

## 5. MVC架构映射

| MVC角色 | 前端对应 | 后端对应 |
|---------|----------|----------|
| Model | Pinia Store + API响应数据 | Pydantic Model + 数据文件读写 |
| View | Vue SFC组件（.vue文件） | FastAPI 返回的 JSON/YAML 数据 |
| Controller | Vue Router + 组件内逻辑 | FastAPI 路由处理函数 |
