# 详细设计：交互式需求收集子系统

## 1. 概述

交互式需求收集子系统负责手机模拟器编辑器的完整运行时，包括：

- 模拟器独立 document 生命周期
- 模拟器状态栏
- 组件面板与组件预览
- 节点树渲染
- 拖拽编辑
- 结果序列化

`v0.2.0` 的关键变化是：模拟器不再只是主文档中的视觉区域，而是由 `PhoneFrame` 宿主创建独立 iframe document，并在该 document 中挂载交互式编辑器应用。

## 3. 模块职责

| 模块 | 职责 |
|------|------|
| 模拟器宿主模块 | 创建 iframe、初始化独立 document、挂载编辑器运行时 |
| 模拟器文档应用模块 | 组织状态栏、画布、组件面板和底部 Home Indicator |
| 组件面板模块 | 展示组件与布局列表，提供拖拽源与卡片预览 |
| 节点渲染模块 | 递归渲染 `LayoutNode`，承接容器排序和嵌套拖拽 |
| 组件管理模块 | 在目标 document 中加载 Vant、SFC 样式和运行时组件 |
| 状态与序列化模块 | 管理节点树、编辑器状态并导出结果 |

## 4. 模块间关系

| 提供方 | 消费方 | 说明 |
|--------|--------|------|
| 组件管理模块 | 节点渲染模块 | 提供运行时组件解析能力 |
| 组件管理模块 | 组件面板模块 | 为预览卡片加载运行时组件 |
| 组件面板模块 | 节点渲染模块 | 通过 `vuedraggable` `pull: 'clone'` 将元数据拖入画布 |
| 状态模块 | 模拟器文档应用模块 | 提供 `nodeTree`、`previewMode`、`paletteVisible` |
| 状态模块 | 需求收集子系统 | 提供序列化结果 |

## 5. 拖拽机制

### 5.1 设计原则

1. 组件面板和画布容器位于同一 iframe document 中。
2. 所有拖拽行为都通过 `vuedraggable` 完成。
3. 交互式编辑器不再依赖主文档事件，也不再通过顶层工具栏呼出面板。

### 5.2 拖拽配置

- 面板侧：
  - `group: { name: 'sim', pull: 'clone', put: false }`
- 画布容器侧：
  - `group: 'sim'`
  - 接收拖入并支持排序

这样可以保证：

1. 面板卡片只能拖出，不能拖入。
2. 节点在容器之间可以自由移动和重排。
3. 面板与画布拖拽上下文闭环在同一 document 中。

## 6. 类与组件详细说明

### 6.1 `PhoneFrame`

文件：`src/modules/interactive/components/PhoneFrame.vue`

职责：

1. 作为宿主组件渲染手机外壳。
2. 创建 iframe document。
3. 注入编辑器基础样式。
4. 在 iframe 中挂载 `SimulatorDocumentApp`。
5. 通过共享 Pinia 实例让主应用与 iframe 编辑器共用状态。

关键点：

- `PhoneFrame` 本身不渲染节点树。
- `PhoneFrame` 也不再直接预加载组件库。
- 其核心职责是“宿主与独立 document 初始化”。

### 6.2 `SimulatorDocumentApp`

文件：`src/modules/interactive/components/SimulatorDocumentApp.vue`

职责：

1. 作为 iframe 内的根 Vue 应用。
2. `provide` 运行时 `runtimeContext`。
3. 预加载 `componentList + layoutList` 对应的运行时组件。
4. 组织 `SimulatorTopBar`、画布和 `ComponentPanel`。

### 6.3 `SimulatorTopBar`

文件：`src/modules/interactive/components/SimulatorTopBar.vue`

职责：

1. 显示模拟器内部状态栏。
2. 提供 `Palette` 开关按钮。
3. 提供 `Preview / Edit` 切换按钮。

说明：

- 组件面板入口从 `AppHeader` 迁移至此。
- 这使编辑器内部控制权回归交互式子系统。

### 6.4 `ComponentPanel`

文件：`src/modules/interactive/components/ComponentPanel.vue`

职责：

1. 显示组件和布局两个标签页。
2. 将 `ComponentInfo` 转为可拖入画布的 `LayoutNode`。
3. 渲染预览卡片 `PanelPreview`。
4. 与 `SimNode` 容器共享同一 `vuedraggable` group。

状态依赖：

- 从 `appStore` 读取 `componentList` / `layoutList`
- 从 `simStore` 读取 `paletteVisible` / `previewMode`

与旧版区别：

- 不再使用 `appStore.panelOpen`
- 不再依赖主文档 `position: fixed`
- 面板现在作为模拟器文档的一部分参与布局

### 6.5 `SimNode`

文件：`src/modules/interactive/components/SimNode.vue`

职责：

1. 递归渲染节点树。
2. 判断节点是否为容器。
3. 在目标 `runtimeContext` 中解析运行时组件。
4. 为容器节点挂载 `vuedraggable`。

属性规范化规则：

| 条件 | 处理 |
|------|------|
| `items` 为字符串 | 以逗号切分为数组 |
| `PropDef.type === 'number'` | 转为 `Number` |
| `PropDef.type === 'boolean'` | 解析布尔值 |
| 其他 | 原值透传 |

### 6.6 `componentLoader`

文件：`src/modules/interactive/services/componentLoader.js`

职责：

1. 创建运行时上下文 `runtimeContext`
2. 在目标 document 中加载 Vant CSS 和 JS
3. 在目标 document 中注入运行时全局样式
4. 用 `vue3-sfc-loader` 按上下文编译 `.vue` 文件
5. 以“上下文 + 路径”为粒度缓存运行时组件

`v0.2.0` 的核心改造在于：

- 组件加载器不再默认绑定主文档
- 所有样式注入、Vant 加载和 SFC 编译都以 `runtimeContext` 为边界

### 6.7 `useSimulatorStore`

文件：`src/stores/simulator.js`

职责：

1. 管理节点树 `nodeTree`
2. 记录当前交互式问题 `currentQuestionId`
3. 管理选中状态 `selectedNodeId`
4. 管理内部编辑器状态 `previewMode` / `paletteVisible`
5. 提供 `loadDefault` / `addNode` / `moveNode` / `removeNode` / `getResult`

## 7. 对外接口

本子系统对外暴露以下接口：

| 接口 | 实现位置 | 说明 |
|------|----------|------|
| `I_Simulator` | `PhoneFrame` + `useSimulatorStore` | 打开交互式问题、加载默认布局、读取结果 |
| `I_ComponentRegistry` | `componentLoader.js` + `appStore` | 加载组件清单并解析运行时组件 |
| `I_LayoutRegistry` | `useSimulatorStore` + `appStore` | 管理默认布局与布局元数据 |

## 9. v0.2.0 设计结论

本次重构后，交互式需求收集子系统具备如下特征：

1. 运行时边界清晰，以独立 document 为编辑器边界。
2. 组件面板、拖拽、运行时组件、Vant 弹层处于同一 document 中。
3. 界面子系统只负责宿主，不再介入编辑器内部控制。
4. 更符合单一职责、接口隔离和依赖倒置原则。
