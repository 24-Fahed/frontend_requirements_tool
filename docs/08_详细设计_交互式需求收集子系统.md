# 详细设计：交互式需求收集子系统

## 1. 概述

交互式需求收集子系统负责手机模拟界面的渲染和拖拽交互操作。包含5个模块：模拟器模块、拖拽控制模块、组件管理模块、布局管理模块、结果序列化模块。

## 2. 模块图

![交互式需求收集子系统_模块图](images/交互式需求收集子系统_模块图.png)

### 模块职责

| 模块 | 职责 |
|------|------|
| 模拟器模块 | 手机外壳CSS渲染，内部组件动态渲染和节点树管理 |
| 拖拽控制模块 | 管理从抽屉到模拟器的拖拽操作，控制添加/移动/删除 |
| 组件管理模块 | 加载和注册开发者提供的组件描述及组件库文件 |
| 布局管理模块 | 加载布局定义，解析布局节点树，提供默认方案 |
| 结果序列化模块 | 将模拟器编辑状态序列化为结构化JSON |

### 模块间关系

| 提供方 | 消费方 | 说明 |
|--------|--------|------|
| 组件管理模块 | 模拟器模块 | 提供组件定义供渲染器动态加载 |
| 布局管理模块 | 模拟器模块 | 提供布局定义和默认设计方案 |
| 拖拽控制模块 | 模拟器模块 | 控制节点树的添加、移动、删除 |
| 结果序列化模块 | 模拟器模块 | 读取节点树并序列化输出 |

## 4. 类详细说明

### 4.1 模拟器模块

#### PhoneFrame

手机外壳组件，纯CSS实现手机外观。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| width | number | 模拟器宽度，默认375px |
| height | number | 模拟器高度，默认667px |
| statusBarText | string | 顶部状态栏文字 |
| render() | VNode | 渲染手机外壳VNode |

#### SimulatorRenderer

核心渲染器，管理布局节点树并动态渲染组件。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| nodeTree | LayoutNode[] | 当前布局节点树 |
| componentMap | Map | 已加载组件的映射 |
| setNodeTree(nodes) | void | 设置节点树（加载默认设计时使用） |
| addNode(parentId, node) | void | 向指定父节点添加子节点 |
| removeNode(nodeId) | void | 删除指定节点 |
| moveNode(nodeId, targetParentId, index) | void | 移动节点到目标父节点的指定位置 |
| updateNodeProps(nodeId, props) | void | 更新节点属性 |
| getNodeTree() | LayoutNode[] | 获取当前节点树 |
| render() | VNode | 递归渲染节点树为VNode |

### 4.2 拖拽控制模块

#### DragDropController

拖拽控制器，协调拖拽操作的完整流程。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| sourceType | string | 当前拖拽源类型（component/layout） |
| sourceData | object | 当前拖拽源数据 |
| startDrag(type, data) | void | 开始拖拽，记录来源信息 |
| endDrag() | void | 结束拖拽，清除状态 |
| onDrop(targetId, index) | void | 处理放置事件，调用Renderer添加节点 |

#### DropZoneManager

放置区域管理器，校验拖拽目标是否合法。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| zones | Map | 已注册的放置区域 |
| registerZone(containerId, maxChildren) | void | 注册一个放置区域 |
| unregisterZone(containerId) | void | 取消注册 |
| isValidDrop(targetId) | boolean | 校验目标区域是否可放置 |
| getDropIndex(event) | number | 根据拖拽事件计算放置索引位置 |

### 4.3 组件管理模块

#### ComponentRegistry

组件注册表，管理所有开发者提供的组件描述。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| components | Map | 已注册组件映射（name → ComponentInfo） |
| registerAll(config) | void | 批量注册组件（从components.json加载） |
| get(name) | ComponentInfo | 按名称获取组件信息 |
| getAll() | ComponentInfo[] | 获取全部组件列表 |
| getByCategory(category) | ComponentInfo[] | 按分类筛选组件 |

#### ComponentLoader

组件库加载器，动态加载开发者提供的组件库文件。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| loadedLibs | Map | 已加载的组件库映射（sourceFile → lib对象） |
| loadLibrary(sourceFile) | Promise | 异步加载组件库JS文件 |
| resolveComponent(componentTag) | Component | 根据标签名从库中解析出Vue组件 |

#### ComponentInfo

组件描述信息数据类。

| 属性 | 类型 | 说明 |
|------|------|------|
| name | string | 组件唯一标识名 |
| label | string | 显示名称 |
| category | string | 分类（基础组件/组合组件） |
| icon | string | 图标标识 |
| props | PropDef[] | 属性定义列表 |
| defaultProps | object | 默认属性值 |
| sourceFile | string | 库文件路径 |
| componentTag | string | 注册标签名 |

#### PropDef

属性定义数据类。

| 属性 | 类型 | 说明 |
|------|------|------|
| name | string | 属性名 |
| type | string | 类型（string/number/boolean/enum） |
| options | string[] | 枚举值（type=enum时） |
| default | any | 默认值 |
| label | string | 显示名称 |

### 4.4 布局管理模块

#### LayoutRegistry

布局注册表，管理所有布局定义。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| layouts | Map | 已注册布局映射（name → LayoutInfo） |
| registerAll(config) | void | 批量注册布局（从layouts.json加载） |
| get(name) | LayoutInfo | 按名称获取布局信息 |
| getAll() | LayoutInfo[] | 获取全部布局列表 |
| getByCategory(category) | LayoutInfo[] | 按分类筛选布局 |

#### LayoutResolver

布局解析器，处理默认设计方案和布局节点操作。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| resolve(defaultLayoutId) | LayoutNode[] | 从default_layout.json中解析指定ID的布局方案 |
| createNode(type, props) | LayoutNode | 创建新的布局节点 |
| cloneTree(nodes) | LayoutNode[] | 深拷贝布局节点树 |

#### LayoutInfo

布局描述信息数据类，继承ComponentInfo的属性结构，额外增加容器相关属性。

| 属性 | 类型 | 说明 |
|------|------|------|
| isContainer | boolean | 是否为容器布局 |
| maxChildren | number | 最大子组件数量，-1为不限制 |
| （继承属性） | - | name, label, category, icon, props, defaultProps, sourceFile, componentTag |

### 4.5 结果序列化模块

#### ResultSerializer

编辑结果序列化器。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| serialize(nodeTree) | object | 将节点树序列化为可存储的JSON结构 |
| deserialize(data) | LayoutNode[] | 将JSON数据反序列化为节点树 |

### 4.6 通用数据类

#### LayoutNode

布局节点，构成模拟器中的组件树。

| 属性 | 类型 | 说明 |
|------|------|------|
| id | string | 节点唯一ID |
| type | string | 组件/布局类型名（对应ComponentInfo.name或LayoutInfo.name） |
| props | object | 当前属性值 |
| children | LayoutNode[] | 子节点列表（容器布局才有） |

## 5. 模块对外接口

本子系统对外暴露 `I_Simulator`、`I_ComponentRegistry`、`I_LayoutRegistry` 三个接口（见子系统接口文档），各模块与接口的映射关系：

| 对外接口 | 实现模块 | 实现类 |
|----------|----------|--------|
| I_Simulator | 模拟器模块 + 结果序列化模块 | SimulatorRenderer + ResultSerializer |
| I_ComponentRegistry | 组件管理模块 | ComponentRegistry |
| I_LayoutRegistry | 布局管理模块 | LayoutRegistry |
