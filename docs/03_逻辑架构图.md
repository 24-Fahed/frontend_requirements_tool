# 需求收集工具 - 逻辑架构图

## 1. 概述

系统采用前后端分离架构：

- 前端：Vue 3 + Pinia + Element Plus + `vuedraggable`
- 后端：Python FastAPI

前端按职责划分为三个子系统：

1. 界面子系统
2. 交互式需求收集子系统
3. 需求收集子系统

## 3. 子系统划分

### 3.1 界面子系统

负责整体页面布局和全局导航控制。

| 模块 | 职责 |
|------|------|
| 顶部工具栏 | 应用标题、侧栏显隐、模拟器显隐、结果保存 |
| 侧边栏 | 需求分类切换 |
| 内容展示区 | 承载问题表格和答案收集界面 |
| 模拟器宿主区 | 承载交互式编辑器宿主组件 |

### 3.2 交互式需求收集子系统

负责手机模拟器编辑器本身。

| 模块 | 职责 |
|------|------|
| 模拟器宿主 | 创建并维护独立 iframe document |
| 模拟器文档应用 | 在独立 document 中渲染状态栏、画布、组件面板 |
| 组件面板 | 展示组件和布局列表，提供拖拽源 |
| 模拟器画布 | 渲染节点树，承接容器拖拽 |
| 组件加载器 | 在目标 document 中加载 Vue SFC、Vant 和运行时样式 |
| 结果序列化 | 从节点树导出结构化 JSON |

### 3.3 需求收集子系统

负责需求配置加载、表格展示、答案收集和结果提交。

| 模块 | 职责 |
|------|------|
| 配置加载器 | 获取需求配置、组件配置、默认布局 |
| 表格渲染器 | 渲染问题表格 |
| 回答收集器 | 收集 text / option / interactive 三类答案 |
| 结果导出器 | 提交最终结果到后端 |

## 4. 子系统间交互关系

| 调用方 | 被调用方 | 方式 | 说明 |
|--------|----------|------|------|
| 界面子系统 | 交互式需求收集子系统 | 前端组件嵌入 | `SimulatorArea` 挂载 `PhoneFrame` 宿主 |
| 界面子系统 | 需求收集子系统 | 前端组件嵌入 | `ContentArea` 承载问题表格 |
| 需求收集子系统 | 交互式需求收集子系统 | Store / 组件调用 | 打开交互式问题并读取模拟器结果 |
| 前端 | 后端 | HTTP API | 加载配置、保存结果、获取静态组件文件 |

## 5. 本版架构变化（v0.2.0）

相较于早期版本，`v0.2.0` 有以下关键变化：

1. 组件面板不再由顶部工具栏控制。
2. 组件面板迁入模拟器独立 document，由模拟器状态栏按钮控制。
3. 面板拖拽与画布拖拽在同一 document 中闭环执行。
4. 运行时组件、Vant 样式和弹层行为都以模拟器 document 为边界。

这使界面子系统与交互式子系统的运行时边界与设计边界保持一致。
