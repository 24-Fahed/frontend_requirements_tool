# 详细设计：需求收集子系统

## 1. 概述

需求收集子系统分为前端部分和后端部分。前端负责配置加载、表格渲染、回答收集和结果导出；后端负责文件解析、需求存储、静态资源服务和数据模型校验。

## 2. 模块图

![需求收集子系统_模块图](images/需求收集子系统_模块图.png)

### 前端模块

| 模块 | 职责 |
|------|------|
| 配置加载模块 | 通过API获取后端解析后的配置数据，支持缓存 |
| 表格渲染模块 | 将需求数据渲染为可交互表格，处理文字输入、选项选择、交互式触发 |
| 回答收集模块 | 收集三种类型的回答（text/option/interactive），桥接交互式子系统 |
| 结果导出模块 | 汇总回答，导出为JSON/YAML，提交到后端存储 |

### 后端模块

| 模块 | 职责 |
|------|------|
| 文件解析模块 | 解析开发者提供的YAML配置和JSON组件/布局文件 |
| 需求存储模块 | 将用户需求结果写入YAML和JSON文件 |
| 静态资源模块 | 提供开发者组件库文件的静态访问 |
| 数据模型模块 | Pydantic数据模型，用于请求响应校验 |

## 4. 前端类详细说明

### 4.1 配置加载模块

#### ConfigLoader

配置数据加载器，封装API调用并提供缓存。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| apiClient | ApiClient | API客户端实例 |
| cache | Map | 数据缓存，避免重复请求 |
| loadRequirements(categoryId?) | Promise | 加载需求数据，可指定分类ID |
| loadComponents() | Promise | 加载组件描述配置 |
| loadDefaultLayout(layoutId) | Promise | 加载指定ID的默认布局方案 |
| clearCache() | void | 清除缓存 |

### 4.2 表格渲染模块

#### TableRenderer

需求表格渲染器，管理当前分类下的表格展示。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| currentCategory | Category | 当前选中分类 |
| questions | Question[] | 当前分类的问题列表 |
| setCategory(category) | void | 切换分类，加载对应问题 |
| render() | VNode | 渲染表格 |
| refresh() | void | 刷新表格（回答变化后） |

#### QuestionRow

单行问题渲染组件，处理不同类型的回答交互。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| question | Question | 当前问题数据 |
| answer | Answer | 当前回答 |
| renderInput() | VNode | 根据type渲染不同的输入控件 |
| onTextChange(text) | void | 文字输入回调 |
| onOptionSelect(options) | void | 选项选择回调 |
| onInteractiveTrigger() | void | 触发交互式收集回调 |

### 4.3 回答收集模块

#### AnswerCollector

回答收集器，管理所有问题的回答数据。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| answers | Map | 已收集的回答映射（questionId → Answer） |
| collect(questionId, answer) | void | 收集一条回答 |
| get(questionId) | Answer | 获取指定问题的回答 |
| getAll() | Answer[] | 获取全部回答 |
| remove(questionId) | void | 删除指定回答 |
| clear() | void | 清空所有回答 |

#### InteractiveBridge

交互式需求收集的桥接器，连接需求收集与交互式模拟器。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| simulator | I_Simulator | 模拟器接口实例 |
| currentQuestionId | string | 当前关联的问题ID |
| openSimulator(questionId, defaultLayoutId?) | void | 打开模拟器，加载指定默认布局 |
| saveSimulatorResult() | LayoutData | 保存模拟器编辑结果并返回 |
| closeSimulator() | void | 关闭模拟器 |

### 4.4 结果导出模块

#### ResultExporter

结果导出器，负责汇总和提交。

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| apiClient | ApiClient | API客户端实例 |
| exportAsJson(answers) | object | 将回答列表导出为JSON结构 |
| exportAsYaml(answers) | string | 将回答列表导出为YAML字符串 |
| submit(answers) | Promise | 将结果提交到后端API |

### 4.5 前端数据类

#### Answer

| 属性 | 类型 | 说明 |
|------|------|------|
| questionId | string | 问题ID |
| categoryId | string | 分类ID |
| type | string | 回答类型（text/option/interactive） |
| value | string \| string[] \| LayoutData | 回答内容 |
| customValue | string | 自定义输入值（allowCustom时） |

## 5. 后端类详细说明

### 5.1 数据模型模块

#### RequirementDataModel

需求配置的Pydantic顶层模型。

| 属性 | 类型 | 说明 |
|------|------|------|
| project | ProjectModel | 项目信息 |
| categories | list[CategoryModel] | 需求分类列表 |

#### CategoryModel

| 属性 | 类型 | 说明 |
|------|------|------|
| id | str | 分类ID |
| name | str | 分类名称 |
| order | int | 排序序号 |
| questions | list[QuestionModel] | 问题列表 |

#### QuestionModel

| 属性 | 类型 | 说明 |
|------|------|------|
| id | str | 问题ID |
| text | str | 问题内容 |
| purpose | str | 问题目的 |
| type | str | 回答类型 |
| options | list[str] \| None | 选项列表 |
| allow_custom | bool | 是否允许自定义输入 |
| interactive_config | dict \| None | 交互式配置 |
| note | str \| None | 备注 |

#### AnswerModel

| 属性 | 类型 | 说明 |
|------|------|------|
| category_id | str | 分类ID |
| question_id | str | 问题ID |
| type | str | 回答类型 |
| value | Any | 回答内容 |
| custom_value | str \| None | 自定义值 |

#### ResultModel

| 属性 | 类型 | 说明 |
|------|------|------|
| project | ProjectModel | 项目信息 |
| submitted_at | str | 提交时间 |
| answers | list[AnswerModel] | 回答列表 |

### 5.2 文件解析模块

#### YamlParser

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| parse_requirements(file_path) | RequirementDataModel | 解析YAML需求配置文件，返回校验后的数据模型 |

#### JsonParser

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| parse_components(file_path) | dict | 解析组件描述JSON文件 |
| parse_layouts(file_path) | dict | 解析布局定义JSON文件 |
| parse_default_layout(file_path, layout_id) | dict | 解析默认设计方案中指定ID的布局 |

### 5.3 需求存储模块

#### ResultStorage

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| save(result, output_dir) | void | 将结果同时写入YAML和JSON文件 |

#### YamlWriter

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| write_yaml(data, file_path) | void | 写入YAML格式文件（人类可读） |
| write_json(data, file_path) | void | 写入JSON格式文件（结构化） |

### 5.4 静态资源模块

#### StaticFileServer

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| get_file(path) | FileResponse | 返回指定路径的静态文件 |
| list_files(dir_path) | list[str] | 列出目录下的文件列表 |

## 6. 模块对外接口

本子系统对外暴露的HTTP接口（见子系统接口文档）：

| 对外接口 | 实现模块 | 说明 |
|----------|----------|------|
| I_ParseService (GET /api/requirements) | 文件解析模块 | YamlParser.parse_requirements |
| I_ParseService (GET /api/components) | 文件解析模块 | JsonParser.parse_components |
| I_ParseService (GET /api/default-layout) | 文件解析模块 | JsonParser.parse_default_layout |
| I_StorageService (POST /api/results) | 需求存储模块 | ResultStorage.save |
| I_StorageService (GET /api/results) | 需求存储模块 | YamlWriter 读取 |
| I_StaticService (GET /static/lib/{path}) | 静态资源模块 | StaticFileServer.get_file |
