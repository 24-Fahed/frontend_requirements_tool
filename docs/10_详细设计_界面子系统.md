# 详细设计：界面子系统

## 1. 概述

界面子系统负责工具的整体布局与全局导航，不负责交互式编辑器内部行为。

`v0.2.0` 起，组件面板入口从顶部工具栏移除，组件面板完全归属交互式需求收集子系统。

## 2. 模块图

![界面子系统_模块图](images/界面子系统_模块图.png)

## 3. 模块职责

| 模块 | 职责 |
|------|------|
| 顶部工具栏模块 | 标题显示、侧栏显隐、模拟器显隐、保存结果 |
| 侧边栏模块 | 显示需求分类并切换激活分类 |
| 内容展示区模块 | 承载需求问题表格 |
| 模拟器区域模块 | 承载交互式编辑器宿主组件 |

## 4. 模块间关系

| 提供方 | 消费方 | 说明 |
|--------|--------|------|
| 顶部工具栏模块 | 侧边栏模块 | 通过共享状态控制侧栏显隐 |
| 侧边栏模块 | 内容展示区模块 | 切换当前激活分类 |
| 内容展示区模块 | 交互式需求收集子系统 | 打开交互式问题时显示模拟器 |
| 模拟器区域模块 | 交互式需求收集子系统 | 挂载 `PhoneFrame` 宿主组件 |

## 5. 组件详细说明

### 5.1 `AppHeader`

文件：`src/modules/ui/components/AppHeader.vue`

职责：

1. 显示项目标题
2. 切换侧边栏显示状态
3. 切换模拟器区域显示状态
4. 提交结果到后端

不再负责：

- 组件面板开关
- 交互式编辑器内部工具入口

### 5.2 `SideBar`

文件：`src/modules/ui/components/SideBar.vue`

职责：

1. 展示分类列表
2. 切换 `activeCategoryId`

### 5.3 `ContentArea`

文件：`src/modules/ui/components/ContentArea.vue`

职责：

1. 根据 `activeCategoryId` 渲染当前问题表格
2. 为交互式问题提供打开模拟器入口

### 5.4 `SimulatorArea`

文件：`src/modules/ui/components/SimulatorArea.vue`

职责：

1. 作为右侧模拟器宿主区域
2. 根据 `simulatorVisible` 控制显隐
3. 渲染 `PhoneFrame`

说明：

- `SimulatorArea` 不再直接渲染 `ComponentPanel`
- `ComponentPanel` 已迁移到 `PhoneFrame` 创建的独立 document 中

## 6. 对外接口

本子系统对外暴露以下接口：

| 接口 | 实现位置 | 说明 |
|------|----------|------|
| `I_Navigation.switchCategory` | `SideBar.selectCategory -> store.setCategory` | 切换分类 |
| `I_Layout.setSimulatorVisible` | `AppHeader.toggleSimulator -> store.simulatorVisible` | 控制模拟器显隐 |
| `I_Layout.setContentData` | `ContentArea.activeCategory` | 提供当前内容数据 |

`v0.2.0` 起，`I_Navigation.togglePanel` 已废弃。

## 7. 实现文件映射

| 模块 | 文件 |
|------|------|
| 顶部工具栏模块 | `src/modules/ui/components/AppHeader.vue` |
| 侧边栏模块 | `src/modules/ui/components/SideBar.vue` |
| 内容展示区模块 | `src/modules/ui/components/ContentArea.vue` |
| 模拟器区域模块 | `src/modules/ui/components/SimulatorArea.vue` |

## 8. v0.2.0 设计结论

重构后，界面子系统重新收敛为“宿主与导航层”：

1. 只负责页面结构和全局操作。
2. 不再控制交互式编辑器内部面板。
3. 与交互式子系统的边界更清晰，更符合单一职责原则。
