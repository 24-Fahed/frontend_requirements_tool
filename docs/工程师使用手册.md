# 需求收集工具 - 工程师使用手册

## 1. 工具简介

本工具帮助你（开发工程师）向客户收集软件需求。你编写配置文件定义问题和可选的 UI 组件，客户在浏览器中通过表格填写和手机模拟器拖拽完成需求表达。

## 2. 你的工作流程

```
编写配置文件 → 部署到服务器 → 客户填写需求 → 取回结果文件
```

## 3. 配置文件结构

你需要准备以下文件，放入服务器的配置目录（默认 `data/dev-config/`）：

```
dev-config/
├── requirements.yaml       # 需求问题定义（必需）
├── components.json          # UI 组件描述（必需）
├── default_layout.json      # 模拟器默认布局（可选）
└── lib/                     # 组件库文件（必需）
    └── MyComponent/
        └── MyComponent.vue
```

## 4. requirements.yaml 编写指南

### 4.1 基本结构

```yaml
project:
  id: "项目英文标识"
  name: "项目名称（显示在工具顶部）"
  description: "项目描述（可选）"

categories:
  - id: "分类标识"
    name: "分类名称（显示在侧边栏）"
    order: 1
    questions:
      - id: "问题标识"
        text: "问题内容"
        purpose: "为什么问这个问题"
        type: text        # text | option | interactive
        note: "提示信息（可选）"
```

### 4.2 三种问题类型

#### 文字输入 (text)

客户直接打字回答。

```yaml
- id: "q1"
  text: "您的联系方式？"
  purpose: "用于后续沟通"
  type: text
  note: "手机号或邮箱"
```

#### 选项选择 (option)

客户从你提供的选项中选择。

```yaml
- id: "q2"
  text: "您需要哪些功能？"
  purpose: "确定开发范围"
  type: option
  options:
    - "产品展示"
    - "在线预约"
    - "用户管理"
    - "自定义"
  allowCustom: true        # 允许客户自己填写，默认 false
  note: "可多选"
```

#### 交互式设计 (interactive)

客户在手机模拟器中拖拽组件来布局。

```yaml
- id: "q3"
  text: "请设计首页布局"
  purpose: "了解页面偏好"
  type: interactive
  interactiveConfig:
    defaultLayout: "home-default"    # 引用 default_layout.json 中的布局 ID
  note: "拖动组件到模拟器中设计"
```

`interactiveConfig.defaultLayout` 可选。指定后，客户打开此问题时模拟器会预加载对应的默认布局。

### 4.3 完整示例

```yaml
project:
  id: "shop-app"
  name: "商城小程序需求收集"

categories:
  - id: "info"
    name: "基本信息"
    order: 1
    questions:
      - id: "q1"
        text: "您的店铺名称？"
        purpose: "用于小程序标题"
        type: text
      - id: "q2"
        text: "您需要哪些功能？"
        purpose: "确定开发范围"
        type: option
        options: ["商品展示", "购物车", "订单管理", "自定义"]
        allowCustom: true

  - id: "design"
    name: "页面设计"
    order: 2
    questions:
      - id: "q3"
        text: "请设计首页布局"
        purpose: "了解布局偏好"
        type: interactive
        interactiveConfig:
          defaultLayout: "home-default"
```

## 5. components.json 编写指南

定义模拟器中可供客户拖拽的 UI 组件。

```json
{
  "components": [
    {
      "name": "Button",
      "label": "按钮",
      "category": "基础组件",
      "props": [
        { "name": "text", "type": "string", "default": "按钮", "label": "按钮文字" },
        { "name": "type", "type": "enum", "options": ["primary", "default", "danger"], "default": "default", "label": "类型" }
      ],
      "defaultProps": { "text": "按钮", "type": "default" },
      "sourceFile": "lib/MyButton/MyButton.vue",
      "componentTag": "MyButton"
    }
  ]
}
```

### 字段说明

| 字段 | 必需 | 说明 |
|------|------|------|
| name | 是 | 组件唯一标识，在结果中通过此名称标识 |
| label | 是 | 面板中显示的名称 |
| category | 是 | 分类名，面板按分类展示 |
| props | 是 | 属性定义列表 |
| props[].name | 是 | 属性名 |
| props[].type | 是 | `string`、`number`、`boolean` 或 `enum` |
| props[].options | 条件 | type 为 enum 时必需 |
| props[].default | 是 | 默认值 |
| props[].label | 是 | 属性显示名 |
| defaultProps | 是 | 默认属性值，拖入模拟器时使用 |
| sourceFile | 是 | 组件 `.vue` 文件路径（相对于 dev-config 目录） |
| componentTag | 是 | 组件名称，需与 `.vue` 文件中 `export default` 的 `name` 一致 |
| isContainer | 否 | 设为 `true` 表示可包含子组件 |
| maxChildren | 否 | 最大子组件数量，-1 为不限 |

### 布局容器组件

如果组件可以包含子组件，添加以下字段：

```json
{
  "name": "VStack",
  "label": "纵向布局",
  "category": "布局",
  "isContainer": true,
  "maxChildren": -1,
  "props": [...],
  "defaultProps": {...}
}
```

## 6. default_layout.json 编写指南

为交互式问题提供初始布局，客户可以在此基础上修改。

```json
{
  "layouts": [
    {
      "id": "home-default",
      "name": "首页默认布局",
      "nodes": [
        {
          "id": "root",
          "type": "VStack",
          "props": { "gap": 8, "padding": 8 },
          "children": [
            { "id": "title", "type": "TextBlock", "props": { "content": "欢迎", "fontSize": 18 } },
            { "id": "btn", "type": "Button", "props": { "text": "开始", "type": "primary" } }
          ]
        }
      ]
    }
  ]
}
```

- `id`: 布局 ID，在 requirements.yaml 中通过 `interactiveConfig.defaultLayout` 引用
- `nodes`: 节点树，每个节点有 `id`、`type`（对应 components.json 中的 name）、`props`、`children`（容器组件）

## 7. 组件库文件编写指南（lib/）

组件库文件放在 `lib/` 目录下，采用标准 Vue 单文件组件（SFC）格式。每个组件一个目录，目录名和文件名与组件名一致。

### 7.1 编写惯例

1. **标准 `.vue` SFC 文件** — 使用 `<template>`、`<script>`、`<style scoped>` 三段式结构
2. **每个组件独立一个文件** — 不再将所有组件打包为一个 JS 文件
3. **Vant 依赖自动加载** — 组件加载器负责加载 Vant CSS 和 JS，组件中直接 `import` 使用
4. **不加载 Vue** — Vue 运行时由工具主应用提供

### 7.2 文件结构

```
lib/
├── ProductCard/
│   └── ProductCard.vue
├── LocationNavBar/
│   └── LocationNavBar.vue
└── MyButton/
    └── MyButton.vue
```

每个 `.vue` 文件的 `sourceFile` 路径格式为 `lib/{ComponentName}/{ComponentName}.vue`。

### 7.3 纯 HTML/CSS 组件示例

```vue
<template>
  <button class="my-btn">{{ text }}</button>
</template>

<script>
export default {
  name: 'MyButton',
  props: {
    text: { type: String, default: '按钮' }
  }
}
</script>

<style scoped>
.my-btn {
  padding: 8px 16px;
  border: 1px solid #999;
  background: #fff;
}
</style>
```

### 7.4 使用 Vant UI 的组件示例

组件中直接 `import` Vant 组件，不需要手动加载 CDN：

```vue
<template>
  <div class="my-card">
    <van-button :type="type" :size="size">{{ text }}</van-button>
    <van-icon name="star-o" />
  </div>
</template>

<script>
import { Button, Icon } from 'vant'

export default {
  name: 'MyCard',
  components: { 'van-button': Button, 'van-icon': Icon },
  props: {
    text: { type: String, default: '按钮' },
    type: { type: String, default: 'default' },
    size: { type: String, default: 'normal' }
  }
}
</script>

<style scoped>
.my-card {
  padding: 16px;
}
</style>
```

**要点**：
- `import { Button, Icon } from 'vant'` — 由组件加载器的模块缓存解析，不会发起网络请求
- 组件注册使用 Vue 标准局部注册语法：`{ 'van-button': Button }`
- CSS 使用 `<style scoped>` 避免样式污染

### 7.5 带内部状态的组件示例

如果组件需要内部数据（如解析 props 中的列表字符串），使用 `data()` 函数：

```vue
<template>
  <div class="tags">
    <span v-for="(tag, i) in tagsList" :key="i" class="tag">{{ tag }}</span>
  </div>
</template>

<script>
export default {
  name: 'TagList',
  props: {
    tags: { type: String, default: '标签1,标签2' }
  },
  data() {
    const tagsList = String(this.tags || '').split(',')
      .map(s => s.trim()).filter(Boolean)
    return { tagsList }
  }
}
</script>

<style scoped>
.tags { display: flex; gap: 4px; flex-wrap: wrap; }
.tag { background: #eee; padding: 2px 8px; font-size: 12px; }
</style>
```

### 7.6 sourceFile 路径必须与文件位置一致

`components.json` 中的 `sourceFile` 必须指向实际的 `.vue` 文件路径：

```json
// components.json
{
  "name": "MyButton",
  "sourceFile": "lib/MyButton/MyButton.vue",
  "componentTag": "MyButton"
}
```

```vue
<!-- lib/MyButton/MyButton.vue -->
<template>...</template>
<script>
export default { name: 'MyButton', ... }
</script>
```

## 8. 获取客户需求结果

客户点击"保存结果"后，结果写入输出目录（默认 `data/output/`）：

```
output/
├── result.yaml       # 人类可读的需求汇总
└── result.json       # 完整结构化数据（含交互式布局）
```

**YAML 文件**方便你快速浏览，交互式类型的结果会标注"详见 result.json"。

**JSON 文件**包含完整数据，适合程序处理。交互式布局的完整节点树仅保存在 JSON 中。

## 9. 配置文件检查清单

启动前请确认：

- [ ] `requirements.yaml` 存在且 YAML 语法正确
- [ ] `components.json` 存在且 JSON 语法正确
- [ ] `lib/` 目录存在且包含组件 `.vue` 文件
- [ ] 每个 question 的 type 只能是 `text`、`option`、`interactive`
- [ ] type 为 option 的问题必须有 `options` 字段
- [ ] type 为 interactive 的问题引用的 defaultLayout ID 必须在 default_layout.json 中存在
- [ ] 每个 question 的 id 在同一分类内唯一
- [ ] 每个 category 的 id 全局唯一
- [ ] components.json 中每个组件的 name 唯一
- [ ] sourceFile 指向的 `.vue` 文件在 lib/ 目录下存在
- [ ] `.vue` 文件中使用标准 Vue SFC 格式（`<template>` + `<script>` + `<style scoped>`）
- [ ] 如果使用 Vant 组件，通过 `import { Xxx } from 'vant'` 引入

## 10. 常见问题

**Q: 后端启动报"需求配置文件不存在"**
A: 检查 `data/dev-config/requirements.yaml` 文件是否存在。

**Q: 组件面板中没有组件显示**
A: 检查 `components.json` 格式是否正确，必须包含 `components` 数组。

**Q: 组件面板显示了组件名但无法渲染预览**
A: 打开浏览器控制台检查：(1) `window.vant` 是否存在（Vant 是否加载成功）；(2) 是否有模板编译错误。如果使用了外部 UI 库，确认 CDN 地址正确且可访问。

**Q: 交互式问题打开模拟器后没有默认布局**
A: 检查 `default_layout.json` 中是否存在对应的布局 ID，`interactiveConfig.defaultLayout` 的值是否匹配。

**Q: 客户之前的结果被覆盖了**
A: 每次保存会覆盖结果文件。如需保留历史，请在保存前备份 `data/output/` 目录。

**Q: 如何更换一批全新的需求问题**
A: 替换 `data/dev-config/` 中的配置文件，然后重启容器即可。不需要重新构建镜像。
