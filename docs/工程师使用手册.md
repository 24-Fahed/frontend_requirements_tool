# 需求收集工具 - 工程师使用手册

## 1. 工具简介

本工具帮助你（开发工程师）向客户收集软件需求。你编写配置文件定义问题和可选的 UI 组件，客户在浏览器中通过表格填写和手机模拟器拖拽完成需求表达。

## 2. 你的工作流程

```
编写配置文件 → 部署到服务器 → 客户填写需求 → 取回结果文件
```

## 3. 配置文件结构

你需要准备以下文件，放入服务器的配置目录（默认 `data/dev-config/`）：

```
dev-config/
├── requirements.yaml       # 需求问题定义（必需）
├── components.json          # UI 组件描述（必需）
├── default_layout.json      # 模拟器默认布局（可选）
└── lib/                     # 组件库文件（必需）
    └── your-lib.js
```

## 4. requirements.yaml 编写指南

### 4.1 基本结构

```yaml
project:
  id: "项目英文标识"
  name: "项目名称（显示在工具顶部）"
  description: "项目描述（可选）"

categories:
  - id: "分类标识"
    name: "分类名称（显示在侧边栏）"
    order: 1
    questions:
      - id: "问题标识"
        text: "问题内容"
        purpose: "为什么问这个问题"
        type: text        # text | option | interactive
        note: "提示信息（可选）"
```

### 4.2 三种问题类型

#### 文字输入 (text)

客户直接打字回答。

```yaml
- id: "q1"
  text: "您的联系方式？"
  purpose: "用于后续沟通"
  type: text
  note: "手机号或邮箱"
```

#### 选项选择 (option)

客户从你提供的选项中选择。

```yaml
- id: "q2"
  text: "您需要哪些功能？"
  purpose: "确定开发范围"
  type: option
  options:
    - "产品展示"
    - "在线预约"
    - "用户管理"
    - "自定义"
  allowCustom: true        # 允许客户自己填写，默认 false
  note: "可多选"
```

#### 交互式设计 (interactive)

客户在手机模拟器中拖拽组件来布局。

```yaml
- id: "q3"
  text: "请设计首页布局"
  purpose: "了解页面偏好"
  type: interactive
  interactiveConfig:
    defaultLayout: "home-default"    # 引用 default_layout.json 中的布局 ID
  note: "拖动组件到模拟器中设计"
```

`interactiveConfig.defaultLayout` 可选。指定后，客户打开此问题时模拟器会预加载对应的默认布局。

### 4.3 完整示例

```yaml
project:
  id: "shop-app"
  name: "商城小程序需求收集"

categories:
  - id: "info"
    name: "基本信息"
    order: 1
    questions:
      - id: "q1"
        text: "您的店铺名称？"
        purpose: "用于小程序标题"
        type: text
      - id: "q2"
        text: "您需要哪些功能？"
        purpose: "确定开发范围"
        type: option
        options: ["商品展示", "购物车", "订单管理", "自定义"]
        allowCustom: true

  - id: "design"
    name: "页面设计"
    order: 2
    questions:
      - id: "q3"
        text: "请设计首页布局"
        purpose: "了解布局偏好"
        type: interactive
        interactiveConfig:
          defaultLayout: "home-default"
```

## 5. components.json 编写指南

定义模拟器中可供客户拖拽的 UI 组件。

```json
{
  "components": [
    {
      "name": "Button",
      "label": "按钮",
      "category": "基础组件",
      "props": [
        { "name": "text", "type": "string", "default": "按钮", "label": "按钮文字" },
        { "name": "type", "type": "enum", "options": ["primary", "default", "danger"], "default": "default", "label": "类型" }
      ],
      "defaultProps": { "text": "按钮", "type": "default" },
      "sourceFile": "lib/your-lib.js",
      "componentTag": "MyButton"
    }
  ]
}
```

### 字段说明

| 字段 | 必需 | 说明 |
|------|------|------|
| name | 是 | 组件唯一标识，在结果中通过此名称标识 |
| label | 是 | 面板中显示的名称 |
| category | 是 | 分类名，面板按分类展示 |
| props | 是 | 属性定义列表 |
| props[].name | 是 | 属性名 |
| props[].type | 是 | `string`、`number`、`boolean` 或 `enum` |
| props[].options | 条件 | type 为 enum 时必需 |
| props[].default | 是 | 默认值 |
| props[].label | 是 | 属性显示名 |
| defaultProps | 是 | 默认属性值，拖入模拟器时使用 |
| sourceFile | 是 | 库文件路径（相对于 dev-config 目录） |
| componentTag | 是 | 组件在库文件中的注册名 |
| isContainer | 否 | 设为 `true` 表示可包含子组件 |
| maxChildren | 否 | 最大子组件数量，-1 为不限 |

### 布局容器组件

如果组件可以包含子组件，添加以下字段：

```json
{
  "name": "VStack",
  "label": "纵向布局",
  "category": "布局",
  "isContainer": true,
  "maxChildren": -1,
  "props": [...],
  "defaultProps": {...}
}
```

## 6. default_layout.json 编写指南

为交互式问题提供初始布局，客户可以在此基础上修改。

```json
{
  "layouts": [
    {
      "id": "home-default",
      "name": "首页默认布局",
      "nodes": [
        {
          "id": "root",
          "type": "VStack",
          "props": { "gap": 8, "padding": 8 },
          "children": [
            { "id": "title", "type": "TextBlock", "props": { "content": "欢迎", "fontSize": 18 } },
            { "id": "btn", "type": "Button", "props": { "text": "开始", "type": "primary" } }
          ]
        }
      ]
    }
  ]
}
```

- `id`: 布局 ID，在 requirements.yaml 中通过 `interactiveConfig.defaultLayout` 引用
- `nodes`: 节点树，每个节点有 `id`、`type`（对应 components.json 中的 name）、`props`、`children`（容器组件）

## 7. 组件库文件编写指南（lib/）

组件库文件放在 `lib/` 目录下，由加载器动态加载。

### 7.1 编写惯例

每个库文件必须遵循以下惯例：

1. **IIFE 包裹** — 整个文件是一个自执行函数
2. **自行加载 CDN 依赖** — 如果组件依赖外部 UI 库（如 Vant），由库文件负责加载其 CSS 和 JS
3. **依赖就绪后调用 `__rtRegister(components)`** — 将组件注册表传给加载器
4. **每个组件是标准 Vue 组件选项对象** — `template` + `props` + `components` + `data` + `methods`，以及扩展的 `style` 字段
5. **不加载 Vue** — `window.Vue` 已由主应用提供，无需自行加载

### 7.2 无外部依赖的库文件

如果组件不依赖任何外部 UI 库（纯 HTML/CSS），可以直接同步注册：

```javascript
(function () {
  __rtRegister({
    MyButton: {
      template: '<button class="my-btn">{{ text }}</button>',
      props: ['text'],
      style: '.my-btn { padding: 8px 16px; border: 1px solid #999; background: #fff; }'
    }
  });
})();
```

### 7.3 使用 Vant UI 的库文件

如果组件使用 Vant 4，需要先从 CDN 加载 Vant 的 CSS 和 JS，加载完成后注册：

```javascript
(function () {
  // 1. 加载 Vant CSS
  var cssLink = document.createElement('link');
  cssLink.rel = 'stylesheet';
  cssLink.href = 'https://unpkg.com/vant@4/lib/index.css';
  document.head.appendChild(cssLink);

  // 2. 加载 Vant JS（UMD，自动使用 window.Vue）
  var vantScript = document.createElement('script');
  vantScript.src = 'https://unpkg.com/vant@4/lib/vant.min.js';
  vantScript.onload = function () {
    // 3. Vant 就绪后注册组件
    __rtRegister({
      MyButton: {
        template: '<van-button :type="type" :size="size">{{ text }}</van-button>',
        props: ['text', 'type', 'size'],
        components: { 'van-button': vant.Button }
      },
      MyNavBar: {
        template: '<van-nav-bar :title="title" />',
        props: ['title'],
        components: { 'van-nav-bar': vant.NavBar }
      }
    });
  };
  document.head.appendChild(vantScript);
})();
```

**要点**：
- 不需要加载 Vue，主应用已提供 `window.Vue`
- Vant UMD 版本自动检测 `window.Vue` 并使用它
- `vantScript.onload` 保证 Vant 就绪后才注册组件
- `components` 选项是 Vue 的局部注册语法：`{ 'van-button': vant.Button }`

### 7.4 组件选项对象

每个注册的组件是一个标准 Vue 3 组件选项对象：

| 字段 | 必需 | 说明 |
|------|------|------|
| template | 是 | Vue 模板字符串 |
| props | 是 | 属性名数组，如 `['text', 'type']` |
| components | 否 | 局部注册的子组件，如 `{ 'van-button': vant.Button }` |
| data | 否 | 函数，返回组件内部状态，如 `function () { return { active: 0 }; }` |
| computed | 否 | 计算属性 |
| methods | 否 | 方法 |
| style | 否 | CSS 字符串（工具扩展字段），自动注入页面 |

### 7.5 注册名必须与 componentTag 一致

`__rtRegister` 中的每个 key 必须与 `components.json` 中的 `componentTag` 完全一致：

```json
// components.json
{ "componentTag": "MyButton" }
```

```javascript
// lib/your-lib.js
__rtRegister({
  MyButton: { ... }    // key = componentTag
});
```

## 8. 获取客户需求结果

客户点击"保存结果"后，结果写入输出目录（默认 `data/output/`）：

```
output/
├── result.yaml       # 人类可读的需求汇总
└── result.json       # 完整结构化数据（含交互式布局）
```

**YAML 文件**方便你快速浏览，交互式类型的结果会标注"详见 result.json"。

**JSON 文件**包含完整数据，适合程序处理。交互式布局的完整节点树仅保存在 JSON 中。

## 9. 配置文件检查清单

启动前请确认：

- [ ] `requirements.yaml` 存在且 YAML 语法正确
- [ ] `components.json` 存在且 JSON 语法正确
- [ ] `lib/` 目录存在且包含组件库文件
- [ ] 每个 question 的 type 只能是 `text`、`option`、`interactive`
- [ ] type 为 option 的问题必须有 `options` 字段
- [ ] type 为 interactive 的问题引用的 defaultLayout ID 必须在 default_layout.json 中存在
- [ ] 每个 question 的 id 在同一分类内唯一
- [ ] 每个 category 的 id 全局唯一
- [ ] components.json 中每个组件的 name 唯一
- [ ] sourceFile 指向的文件在 lib/ 目录下存在
- [ ] `__rtRegister` 中的 key 与 components.json 中的 componentTag 一致
- [ ] 库文件中不加载 Vue（由主应用提供）
- [ ] 如果使用外部 UI 库（如 Vant），在 CDN 就绪后才调用 `__rtRegister`

## 10. 常见问题

**Q: 后端启动报"需求配置文件不存在"**
A: 检查 `data/dev-config/requirements.yaml` 文件是否存在。

**Q: 组件面板中没有组件显示**
A: 检查 `components.json` 格式是否正确，必须包含 `components` 数组。

**Q: 组件面板显示了组件名但无法渲染预览**
A: 打开浏览器控制台检查：(1) `window.vant` 是否存在（Vant 是否加载成功）；(2) 是否有模板编译错误。如果使用了外部 UI 库，确认 CDN 地址正确且可访问。

**Q: 交互式问题打开模拟器后没有默认布局**
A: 检查 `default_layout.json` 中是否存在对应的布局 ID，`interactiveConfig.defaultLayout` 的值是否匹配。

**Q: 客户之前的结果被覆盖了**
A: 每次保存会覆盖结果文件。如需保留历史，请在保存前备份 `data/output/` 目录。

**Q: 如何更换一批全新的需求问题**
A: 替换 `data/dev-config/` 中的配置文件，然后重启容器即可。不需要重新构建镜像。
