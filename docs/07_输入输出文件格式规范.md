# 需求收集工具 - 输入输出文件格式规范

## 1. 概述

本文档规定系统的输入和输出文件格式。开发者需要按照本规范准备配置文件，系统按照本规范输出用户需求结果。

## 2. 文件总览

| 方向 | 文件名 | 格式 | 提供方 | 用途 |
|------|--------|------|--------|------|
| 输入 | requirements.yaml | YAML | 开发者 | 需求问题定义 |
| 输入 | components.json | JSON | 开发者 | 组件描述定义 |
| 输入 | layouts.json | JSON | 开发者 | 布局定义 |
| 输入 | default_layout.json | JSON | 开发者 | 默认设计方案 |
| 输入 | lib/ | .js / .vue | 开发者 | 组件库文件 |
| 输出 | result.yaml | YAML | 系统 | 人类可读的需求结果 |
| 输出 | result.json | JSON | 系统 | 结构化的需求结果 |

## 3. 输入文件目录结构

开发者按以下结构组织文件，部署到服务器的 `dev-config/` 目录：

```
dev-config/
├── requirements.yaml          # 需求配置文件（必需）
├── components.json            # 组件描述文件（必需）
├── layouts.json               # 布局定义文件（可选）
├── default_layout.json        # 默认设计方案（可选）
└── lib/                       # 组件库目录（必需）
    ├── my-ui-lib.js
    └── ...
```

## 4. 输入文件详细格式

### 4.1 requirements.yaml（需求配置文件）

定义需求问题、分类、选项等内容。YAML格式，支持注释，方便开发者编写。

```yaml
# 项目基本信息
project:
  id: "mini-program-v1"            # 项目标识（唯一）
  name: "小程序需求收集"             # 项目名称
  description: "收集客户的小程序开发需求"  # 项目描述

# 需求分类（对应侧边栏）
categories:
  - id: "basic-info"               # 分类ID
    name: "基本信息"                 # 分类显示名称
    order: 1                        # 排序序号
    questions:                      # 该分类下的问题列表
      - id: "q1"                    # 问题ID（分类内唯一）
        text: "您的企业名称是什么？"    # 问题内容
        purpose: "用于标题和展示"       # 问题目的
        type: text                  # 回答类型：text | option | interactive
        note: "请填写全称"            # 备注（可选）

      - id: "q2"
        text: "您需要哪些功能模块？"
        purpose: "确定开发范围"
        type: option
        options:                    # 选项列表（type=option时必需）
          - "产品展示"
          - "在线预约"
          - "用户管理"
          - "订单系统"
          - "其他"
        note: "可多选，选择'其他'可自行填写"

      - id: "q3"
        text: "请设计您期望的首页布局"
        purpose: "了解客户视觉偏好"
        type: interactive           # 交互式需求收集
        interactiveConfig:
          defaultLayout: "home-default"  # 引用default_layout.json中的布局ID
        note: "从右侧拖动组件到手机界面中进行布局设计"

  - id: "style"
    name: "风格偏好"
    order: 2
    questions:
      - id: "q4"
        text: "您偏好什么主色调？"
        purpose: "确定UI配色方案"
        type: option
        options:
          - "蓝色系"
          - "绿色系"
          - "红色系"
          - "自定义"
        allowCustom: true           # 是否允许自定义输入

      - id: "q5"
        text: "请设计一个示例页面"
        purpose: "了解客户的详细风格偏好"
        type: interactive
        note: "请在模拟器中拖动组件完成页面设计"
```

**字段说明：**

| 字段 | 层级 | 必需 | 说明 |
|------|------|------|------|
| project | 顶层 | 是 | 项目基本信息 |
| project.id | - | 是 | 项目唯一标识 |
| project.name | - | 是 | 项目名称 |
| project.description | - | 否 | 项目描述 |
| categories | 顶层 | 是 | 需求分类列表 |
| categories[].id | - | 是 | 分类唯一标识 |
| categories[].name | - | 是 | 分类显示名称 |
| categories[].order | - | 否 | 排序序号，默认按数组顺序 |
| categories[].questions | - | 是 | 问题列表 |
| questions[].id | - | 是 | 问题唯一标识（分类内唯一） |
| questions[].text | - | 是 | 问题内容 |
| questions[].purpose | - | 是 | 问题目的 |
| questions[].type | - | 是 | 回答类型：text/option/interactive |
| questions[].options | - | 条件 | type=option时必需 |
| questions[].allowCustom | - | 否 | 是否允许自定义输入，默认false |
| questions[].interactiveConfig | - | 条件 | type=interactive时使用 |
| questions[].interactiveConfig.defaultLayout | - | 否 | 引用默认布局ID |
| questions[].note | - | 否 | 备注信息 |

### 4.2 components.json（组件描述文件）

定义开发者提供的UI组件信息。

```json
{
  "components": [
    {
      "name": "Button",
      "label": "按钮",
      "category": "基础组件",
      "icon": "button-icon",
      "props": [
        {
          "name": "text",
          "type": "string",
          "default": "按钮",
          "label": "按钮文字"
        },
        {
          "name": "type",
          "type": "enum",
          "options": ["primary", "default", "warn"],
          "default": "default",
          "label": "按钮类型"
        },
        {
          "name": "size",
          "type": "enum",
          "options": ["small", "medium", "large"],
          "default": "medium",
          "label": "尺寸"
        }
      ],
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyButton"
    },
    {
      "name": "TextBlock",
      "label": "文本块",
      "category": "基础组件",
      "icon": "text-icon",
      "props": [
        {
          "name": "content",
          "type": "string",
          "default": "文本内容",
          "label": "文本内容"
        },
        {
          "name": "fontSize",
          "type": "number",
          "default": 14,
          "label": "字号"
        }
      ],
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyTextBlock"
    },
    {
      "name": "ProductCard",
      "label": "产品卡片",
      "category": "组合组件",
      "icon": "card-icon",
      "props": [
        {
          "name": "title",
          "type": "string",
          "default": "产品名称",
          "label": "产品标题"
        },
        {
          "name": "image",
          "type": "string",
          "default": "",
          "label": "产品图片URL"
        },
        {
          "name": "price",
          "type": "string",
          "default": "¥0.00",
          "label": "价格"
        }
      ],
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyProductCard"
    }
  ]
}
```

**字段说明：**

| 字段 | 必需 | 说明 |
|------|------|------|
| name | 是 | 组件唯一标识名 |
| label | 是 | 组件在组件面板中的显示名称 |
| category | 是 | 组件分类（基础组件/组合组件等） |
| icon | 否 | 组件图标标识 |
| props | 是 | 组件属性定义列表 |
| props[].name | 是 | 属性名 |
| props[].type | 是 | 属性类型：string/number/boolean/enum |
| props[].options | 条件 | type=enum时必需，枚举值列表 |
| props[].default | 是 | 属性默认值 |
| props[].label | 是 | 属性显示名称 |
| sourceFile | 是 | 组件所在的库文件路径（相对于dev-config目录） |
| componentTag | 是 | 组件在库中注册的标签名，用于动态渲染 |

### 4.3 layouts.json（布局定义文件）

定义可用的布局容器。

```json
{
  "layouts": [
    {
      "name": "VStack",
      "label": "纵向布局",
      "category": "布局",
      "icon": "vstack-icon",
      "props": [
        {
          "name": "gap",
          "type": "number",
          "default": 8,
          "label": "间距"
        },
        {
          "name": "padding",
          "type": "number",
          "default": 0,
          "label": "内边距"
        }
      ],
      "isContainer": true,
      "maxChildren": -1,
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyVStack"
    },
    {
      "name": "HStack",
      "label": "横向布局",
      "category": "布局",
      "icon": "hstack-icon",
      "props": [
        {
          "name": "gap",
          "type": "number",
          "default": 8,
          "label": "间距"
        }
      ],
      "isContainer": true,
      "maxChildren": -1,
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyHStack"
    },
    {
      "name": "Grid2Col",
      "label": "两列网格",
      "category": "布局",
      "icon": "grid-icon",
      "props": [
        {
          "name": "gap",
          "type": "number",
          "default": 8,
          "label": "间距"
        }
      ],
      "isContainer": true,
      "maxChildren": -1,
      "sourceFile": "lib/my-ui-lib.js",
      "componentTag": "MyGrid2Col"
    }
  ]
}
```

**字段说明：**

| 字段 | 必需 | 说明 |
|------|------|------|
| isContainer | 是 | 是否为容器布局（true时允许拖入子组件） |
| maxChildren | 是 | 最大子组件数量，-1表示不限制 |
| 其余字段 | - | 与components.json中的字段相同 |

### 4.4 default_layout.json（默认设计方案）

为交互式需求收集提供初始布局方案。

```json
{
  "layouts": [
    {
      "id": "home-default",
      "name": "首页默认布局",
      "nodes": [
        {
          "id": "root",
          "type": "VStack",
          "props": { "gap": 10, "padding": 10 },
          "children": [
            {
              "id": "header-image",
              "type": "Image",
              "props": { "src": "placeholder-banner", "width": "100%", "height": 150 }
            },
            {
              "id": "nav-grid",
              "type": "Grid2Col",
              "props": { "gap": 8 },
              "children": [
                {
                  "id": "nav-1",
                  "type": "Button",
                  "props": { "text": "功能一", "type": "primary" }
                },
                {
                  "id": "nav-2",
                  "type": "Button",
                  "props": { "text": "功能二", "type": "default" }
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

**字段说明：**

| 字段 | 必需 | 说明 |
|------|------|------|
| id | 是 | 布局方案ID，供requirements.yaml中interactiveConfig.defaultLayout引用 |
| name | 是 | 布局方案名称 |
| nodes | 是 | 布局节点树，结构与LayoutData类型一致 |

## 5. 输出文件详细格式

输出文件保存在服务器的 `output/` 目录。

### 5.1 result.yaml（人类可读的需求结果）

```yaml
# 需求收集结果
# 项目：小程序需求收集
# 提交时间：2024-01-15 14:30:00

project:
  id: "mini-program-v1"
  name: "小程序需求收集"

submittedAt: "2024-01-15T14:30:00+08:00"

answers:
  - categoryId: "basic-info"
    categoryName: "基本信息"
    questionId: "q1"
    question: "您的企业名称是什么？"
    type: text
    answer: "广州市银阁视频机械有限公司"

  - categoryId: "basic-info"
    categoryName: "基本信息"
    questionId: "q2"
    question: "您需要哪些功能模块？"
    type: option
    answer:
      - "产品展示"
      - "在线预约"

  - categoryId: "basic-info"
    categoryName: "基本信息"
    questionId: "q3"
    question: "请设计您期望的首页布局"
    type: interactive

    question: "您偏好什么主色调？"

```

**YAML与JSON的关系：**
- YAML文件面向开发者阅读，interactive类型的结果仅显示备注，指向JSON文件
- JSON文件面向程序处理，包含所有类型回答的完整结构化数据
- 两者内容同源，YAML由系统从JSON数据生成

## 6. 输出目录结构

```
output/
├── result.yaml          # 人类可读的需求结果汇总
└── result.json          # 结构化需求结果（完整数据）
```
